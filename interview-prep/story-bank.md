# Story Bank — Master STAR+R Stories

This file accumulates your best interview stories over time. Each evaluation (Block F) adds new stories here. Instead of memorizing 100 answers, maintain 5-10 deep stories that you can bend to answer almost any behavioral question.

## How it works

1. Every time `/career-ops oferta` generates Block F (Interview Plan), new STAR+R stories get appended here
2. Before your next interview, review this file — your stories are already organized by theme
3. The "Big Three" questions can be answered with stories from this bank:
   - "Tell me about yourself" → combine 2-3 stories into a narrative
   - "Tell me about your most impactful project" → pick your highest-impact story
   - "Tell me about a conflict you resolved" → find a story with a Reflection

### [Portfolio] Inventario App como sistema completo
**Source:** Report #001 - Genesys - Desarrollador Full Stack Junior
**S (Situation):** Necesidad de demostrar capacidad de construir una aplicación completa para procesos de inventario, compras y ventas.
**T (Task):** Crear un sistema funcional con datos persistentes, documentos y reportes.
**A (Action):** Desarrollé una aplicación Python con Tkinter, SQLAlchemy, SQLite/PostgreSQL, módulos de compras/ventas, generación de PDFs y exportación CSV.
**R (Result):** Proyecto usable como evidencia de programación, arquitectura por capas, manejo de datos y procesos operativos.
**Reflection:** Para entrevistas TI, este proyecto debe presentarse como prueba de pensamiento de producto y mantenibilidad, no solo como app de escritorio.
**Best for questions about:** proyectos, arquitectura, SQL, aplicaciones CRUD, aprendizaje práctico, iniciativa.

### [Automation] Exelcior Apolo para automatizar Excel
**Source:** Report #001 - Genesys - Desarrollador Full Stack Junior
**S (Situation):** Flujos logísticos y clínicos dependen de archivos Excel repetitivos y propensos a error.
**T (Task):** Crear una herramienta que transforme, valide e imprima archivos de forma más ordenada.
**A (Action):** Construí una app Python con Tkinter, pandas, openpyxl, SQLAlchemy y ReportLab para automatizar validación, transformación, PDFs e impresión.
**R (Result):** Proyecto que demuestra automatización aplicada a procesos reales, manejo de datos y empaquetado para Windows.
**Reflection:** La automatización es más valiosa cuando nace de conocer el proceso operativo completo.
**Best for questions about:** automatización, Python, Excel, reportes, usuarios operativos, mejora de procesos.

### [Full Stack] AMILAB frontend y backend
**Source:** Report #001 - Genesys - Desarrollador Full Stack Junior
**S (Situation):** Necesidad de construir una base web para catálogo B2B/e-commerce de insumos médicos.
**T (Task):** Separar frontend y backend, dejando rutas, componentes, endpoints y estructura lista para integración real.
**A (Action):** Desarrollé frontend con React, TypeScript, Vite y React Router, y backend serverless con TypeScript, Vercel Functions, Firebase/Firestore, endpoints REST, Auth, CORS, Zod, logging y tests.
**R (Result):** Proyecto full stack demostrable con frontend desplegado, backend serverless y una arquitectura alineada con aplicaciones web modernas.
**Reflection:** Este proyecto es la mejor evidencia para roles full stack junior porque muestra interfaz, API, validación, datos, despliegue y separación de responsabilidades.
**Best for questions about:** full stack, React, TypeScript, APIs REST, backend serverless, Firebase, despliegue, arquitectura.
